// Система создания тегов
// Файл - JSON
// внутри JSON-а - список элементов с уникальными айдишниками-полями

// У каждого элемента есть какой-то конечный набор аттрибутов (description, name, image и тд)
// Так же есть тэг
// У тэга два аттрибута - self или apply. Это определяет, куда будет применяться тэг - к самому объекту или к персонажу (который, например, наденет предмет)
// Внутри каждого из них - набор собственно тэгов
// У каждтог тэга есть type и value. Опционально name, target (для мод), conditions, outerConditions, onSuccess, onFail
// Возможные типы: skill, action и mod
// Условия - пишутся либо через and, либо через or
// Существующуюие условия - status / notStatus, knownLore / unknownLore
// Условия проверяются по соответствующему массиву значений, по очереди для каждого значения
// onSuccess / onFail вызывает какие-то резолверы
// Например добавление статуса, знания, событие
// У них так же может быть условие
